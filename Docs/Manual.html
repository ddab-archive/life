Introduction

This program demonstrates the Game of Life on a 21×21 grid.

A bit of background before we get started. This program has roots going back to a DOS program written in 1992. The core of the present program was written in 1997 and 1998, with only slight changes since. The programming is not something the author is proud of, going back as it does to early days of learning object oriented programming.

What's more the small grid is a left over from a time when PCs were so slow that calculating each generation took some appreciable time. This grid size seemed reasonable for the processing power, but it's really much too small.

Actually, the only reason the program is still around is that it's a bit of personal history. Not very good, but the author can't bear to throw it out! However, it won't be developed any further and is not supported. So no questions, suggestions or bug reports please.

You have been warned!

Setting up and running a game

The game starts with an empty grid. "Life forms" are placed on the grid by double clicking the cell where a life form is required or by using the Create button. Life forms can be removed by double clicking the cell again or by using the Kill button.

Once all life forms are placed the game can be started. This is done by clicking the Next Generation button. The program then calculates which life forms die and which are born according the rule that is currently in force. The next generation is then displayed. Successive generations can be displayed by repeatedly clicking the Next Generation button.

Life forms can be added to or removed from the game at any stage.

Information about each generation is displayed in the Generation Info box. Information about the currently highlighted cell is displayed in the Square Info panel. A cell is highlighted by single-clicking it. The cell remains highlighted until another cell is clicked. The Square Info panel is updated whenever the grid changes, either because of a new generation or because life forms are added or removed. When a group of cells are selected this information is not available.

A snapshot of the current generation can be taken by clicking the Snapshot button. The game can be replayed from this snapshot. The snapshot is preserved only until another snapshot is taken, a new game is opened, the game is cleared or the program is halted.

Clicking the Replay button re-runs the game. If a snapshot has been taken you are given the choice of whether to replay from the snapshot or the start of the game. If no snapshot has been taken then the game is always replayed from the start.

The current game can be cleared by clicking the Clear button. This also clears any snapshot.

Saving and loading games

Games can be loaded from and saved to disk.

To save a game click the Save toolbar button. Doing so displays the Save Options dialogue box where you choose whether to save the start of the game or the current generation. If you have taken a snapshot you may also choose to save that. Clicking OK displays a standard Windows Save dialogue box where you choose the file name.

To open a previously saved game, click the Open toolbar button. This displays a standard Windows Open dialogue box from where you can choose the required file.

Rules

The program comes with the ability to run the game using John Conway's Game Of Life rules. However you can also define and use your own.

To do this click the Rules toolbar button to open the Rules dialogue box. Names of available rules are listed in the Rule combo box. Details of the currently selected rule are displayed in the check boxes. You can edit these rules to decide how the game determines whether a new life form is born or an existing life form dies.

The check boxes represent the number of neighbours a cell must have for the rule to be activated. The birth rule check boxes determine how many neighbouring life forms an empty cell must have for a new life form to be born. The survival rule determines how many neighbouring life forms a cell containing a life form must have for the life form to die.

For example the Game Of Life rules specify that if a blank cell has three neighbouring life forms then a new life form is born while a life-form with 2 or 3 neighbouring life forms will die. In this case the birth rule column has only check box 3 checked while the survival rule column has check boxes 2 and 3 checked.

You create your own rules by checking and clearing the check boxes. To get an English description of your rule click the Explain button. Click the Save As button to save a rule you have edited under a new name or click Save to overwrite the existing rule. Clicking Save As brings up a dialogue box where you can name the rule or select an existing rule to overwrite.

The selected rule can be deleted using the Delete button.

To apply a rule to the current game, select it's name in the Rule combo box then click the Apply button. Note that clicking the Close button simply closes the dialogue box – it does not use the selected rule for the current game.

You may not delete or modify the built in Game Of Life rule.

The current rule is displayed in the main window's status bar. Double clicking the text in the status bar has the same effect as clicking the Rules toolbar button.

Custom rules are stored in the program's configuration file, so they persist between sessions.

Colour schemes

The colour scheme used for the game grid can be customised by clicking the Colours toolbar button to display the Colour Schemes dialogue box.

Available schemes can be selected from the Scheme drop-down list. Clicking OK closes the dialogue box and applies the selected scheme to the game grid. Click Cancel to leave the current colour scheme unchanged.

New colour schemes are created using the controls and grid below to drop-down list. Select the element you want to change from the Current element drop-down list then select the desired colour from the Colour drop-down list. Repeat this for each element you want to change.

A short cut to choosing a grid element is to click on the small grid as follows:

  • Click the square containing the life form to select the Life form element.
  • Click the square with the dashed highlight to select the Highlight element.
  • Click any other square to select either the Background or Grid line elements. Clicking the square repeatedly toggles between the two elements.

Notice that as soon as you change the colour of one of the elements the Scheme drop-down list will clear, indicating that you have a new theme that has not yet been named.

Once you have created your scheme click the Save As button. This displays a dialogue box where you can enter a name for the scheme or choose an existing scheme name to overwrite.

You can delete any of your user defined themes by selecting it in the Scheme drop-down list and clicking the Delete button.

Note that the built in Default theme can't be deleted or overwritten.

You don't have to name a scheme – you can simply create it and click OK to apply it. That is useful if you want to see what the scheme looks like in the main window, but it's a good idea to save any scheme you like to avoid accidentally overwriting it.

Colour schemes, including one unnamed scheme, are stored in the program's configuration file and so persist between sessions.

The name of current colour scheme (or "<un-named>") is displayed in the main window's status bar. Double clicking the text in the status bar has the same effect as clicking the Colours toolbar button.

Help

Clicking the Help button displays this document while clicking the About button displays the program's about box.

Exiting the program

To close the program click the Exit toolbar button, click the window's close button or press Alt+F4.

A prompt will be displayed asking you to confirm that you want to close the program. Admittedly that's irritating, but it does give you a chance to go back if you have forgotten to save the game. The program is far too stupid to detect whether or not a game has been saved!
